AlifeGuard: Контроль популяции для STALKER Anomaly, автор Damian
GitHub: https://github.com/damiansirbu-stalker/AlifeGuard
Список изменений: https://github.com/damiansirbu-stalker/AlifeGuard/blob/main/doc/changelog

! Сбросьте настройки MCM при обновлении до новой версии !

Лаги в поздней игре, микрофризы, падение FPS от раздутого A-Life.
Слишком много онлайн-сущностей душат движок X-Ray. AlifeGuard держит их количество
под настраиваемым порогом, деспавня самых дальних NPC первыми. Умные укрытия
заполняются естественным образом. Мир остаётся живым без нагрузки на процессор.

Сбалансированный деспавн:
  Сначала дальние. Все онлайн-сталкеры и мутанты сортируются по расстоянию. NPC рядом с вами всегда уходит последним.
  30-секундная пауза после каждой загрузки уровня даёт миру устояться.
  Однопроходный сбор через game_objects_iter (нативный итератор C++, без аллокации Lua-таблиц).
  Настраиваемый порог и интервал. По умолчанию: 80 максимум, проверка каждые 30 секунд.

Многослойная защита:
  Каждая сущность проходит 4 независимых проверки перед удалением. Любое совпадение оставляет её в живых.
  Слой 1 -- Идентичность: сюжетные персонажи, сюжетные ID отрядов, компаньоны, именные NPC (торговцы, механики, лидеры, медики, бармены, проводники)
  Слой 2 -- Квестодатели: NPC, на которых ссылаются активные задания (по ID NPC или ID отряда)
  Слой 3 -- Награды/заложники: цели, отслеживаемые axr_task_manager
  Слой 4 -- Целевые отряды: штурмовые, наградные, заложные, курьерские отряды

Производительность:
  Покадровый деспавн. Вместо освобождения всех лишних сущностей за один кадр AlifeGuard
  использует отложенную очередь (xslice) для распределения работы по кадрам. Сущности
  освобождаются по одной за кадр. 20 лишних NPC = 20 кадров на очистку. Ровные
  фреймтаймы, без фризов, без подвисаний.
  Сбор использует квадраты расстояний, кэшированные вызовы движка и слабоключевые
  кэши для проверок защиты. Мод собирает и сортирует за один проход, затем очередь
  освобождает в фоне.
  См. doc/img/benchmark_despawn_spread.jpg для измеренных данных.

Чистое освобождение через движок:
  Каждое удаление проходит через alife_release_id, по одному за кадр. Освобождение
  нескольких сущностей за один кадр -- известная причина крашей X-Ray (каскад net_Relcase).
  Покадровый темп полностью это предотвращает. Без фантомных сущностей, без осиротевших
  данных, без утечек памяти.

Кнопки MCM:
  Показать статус     Текущие счётчики сущностей и статистика защиты через КПК
  Принудительная очистка   Немедленно удалить лишние сущности
  Удалить обычных     Удалить все респавнящиеся отряды (умные укрытия заполнятся заново)
  Удалить всех        Удалить ВСЕ отряды, включая сюжетных (может сломать квесты)

Отладочное логирование:
  Переключатель MCM. При включении логирует каждую проверку защиты, каждое освобождение
  с таймингом и расстоянием, и полную строку итогов за цикл. Записывается в alifeguard.log.
  Нулевая нагрузка при отключении (все отладочные вызовы за булевым гейтом).

Уведомления КПК:
  Иммерсивные сообщения при запуске очистки. Переключаются через MCM.

Требования:
Anomaly 1.5.3
Моддед-экзешники
xlibs (https://www.moddb.com/mods/stalker-anomaly/addons/xlibs-1001)
MCM

Установка (MO2):
1. Установите xlibs
2. Установите AlifeGuard
3. Порядок загрузки не имеет значения
4. Настройте через MCM

Удаление (MO2):
Отключите или удалите в MO2.

Совместимость:
Совместим со всеми вариантами моддед-экзешников (Demonized, AOE, MT).
Работает с AlifePlus, ZCP, Warfare, GAMMA и любым другим A-Life или warfare модом. Не модифицирует базовые скрипты. Освобождает сущности только через стандартный API движка.

Известные проблемы:
Крайне редкий краш при освобождении сущности (ассерт Perform_reject).
Это баг движка X-Ray в отслеживании родителя инвентаря -- тот же путь кода, используемый всеми аналогичными модами: Grok Dynamic Despawner, Night Mutants (xcvb), Phantoms (xcvb), Guards Spawner (xcvb), Dynamic Anomalies (Demonized), Boomsticks and Sharpsticks скрипты деспавна (Mich).
Скриптового исправления не существует.

Разработка:
Написано на основе исходного кода X-Ray Monolith, исходного кода моддед-экзешников Demonized и распакованных gamedata Anomaly 1.5.3.
Паттерны кода и использование движка валидированы по работам авторитетных GAMMA-моддеров (Demonized, Vintar0, RavenAscendant, xcvb).
Код валидируется в реальном времени многоступенчатым конвейером: luacheck, selene, tree-sitter AST анализ, контрактные правила, разрешение кросс-файловых зависимостей, анализ цикломатической сложности, обнаружение паттернов крашей и уязвимостей, интеграционное тестирование lua54 со стабами движка X-Ray, сканирование секретов gitleaks.
Полный отчёт в doc/test-report.log.
