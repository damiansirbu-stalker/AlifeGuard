A-Life в поздней игре накапливает слишком много активных сущностей. AI, физика и
поиск пути работают в одном потоке. Производительность падает с ростом количества.
Моды популяции вроде ZCP и Redone усугубляют проблему, увеличивая скорость спавна.
Зомби-сущности от сломанных релизов, осиротевшие члены отрядов и утечки памяти на
уровне движка ухудшают ситуацию со временем.

AlifeGuard контролирует онлайн-популяцию, сохраняя работу A-Life. Отряды остаются
валидными, умные территории работают корректно, нет циклов перерождения. Снижает
нагрузку, не ломая симуляцию.

Простые деспавнеры удаляют отдельных NPC без учёта структуры отрядов. Это может
уничтожить целые отряды, открыть слоты перерождения, оставить умные территории в
нерабочем состоянии, где они перестают спавнить, и помешать модам, управляющим
отрядами. AlifeGuard работает на уровне отрядов и учитывает бухгалтерию спавна движка.

Поотрядная очистка:
  Сущности обрабатываются как часть отряда, а не индивидуально. Рядовые члены
  удаляются первыми. Командиры удаляются только в крайнем случае.
  Это сохраняет отряды в SIMBOARD, держит счётчики already_spawned нетронутыми и
  предотвращает цикл перерождения, который вызывают индивидуальные деспавнеры.
  Отряды, управляемые другими модами (AlifePlus, Warfare, Guards Spawner),
  деприоритизированы: их члены идут после неуправляемых, их командиры -- последними.

Равномерная очистка (round-robin):
  Удаления распределяются между фракциями и типами мутантов. Одна сущность на
  категорию за раунд. Ни одна группа не страдает непропорционально.

Гистерезис:
  Отдельные пороги срабатывания и цели (по умолчанию: срабатывание при 80, очистка
  до 70). Очистка работает циклами, а не постоянно реагирует на малые изменения.

Покадровое освобождение:
  Сущности освобождаются по одной за кадр через xslice (кооперативное разделение
  времени). 30 лишних сущностей = 30 кадров на очистку. Без пиков, без фризов,
  без каскадных крашей net_Relcase.

Защита:
  Сюжетные отряды, торговцы, именные NPC, компаньоны, квестодатели, цели наград и
  заложники никогда не удаляются. Проверки на уровне отряда с TTL-кэшем только
  позитивных результатов. Постчленная проверка ловит именных NPC, которые не являются
  командирами отрядов.

Санитайзер смарт-террейнов:
  Защитный проход, очищающий повреждённые счётчики респавна на смарт-террейнах.
  Отрицательные или нечисловые значения already_spawned вызывают краши сохранения
  (переполнение u8 при STATE_Write) и бесконечные циклы спавна (max > num в гейте
  респавна всегда истина). Санитайзер запускается на actor_on_reinit до начала
  alife-разгона при загрузке, затем каждые 300 секунд во время игры. Обход таблицы
  SIMBOARD.smarts на Lua. Субмиллисекундная стоимость на 50-200 смартов. Переключатель
  и интервал в MCM.

Производительность:
  Однопроходный сбор (нативный итератор C++). Кэшированные проверки защиты.
  Субмиллисекундное сканирование 200 сущностей. 0.05мс на освобождение. Нулевая
  нагрузка отладки при уровне логов < DEBUG.

Совместимость с модами:
  Большинство модов популяции конфликтуют с модами A-Life. AlifeGuard спроектирован
  для сосуществования с AlifePlus, Warfare, ZCP, GAMMA, Guards Spawner и любым модом,
  использующим scripted_target. Скриптованные отряды сохраняются как можно дольше.
  Командиры остаются, поэтому назначения отрядов продолжаются. Счётчики спавна
  остаются консистентными. Отслеживание популяции Warfare остаётся точным.
  Цепочки причин/последствий AlifePlus завершаются, потому что целевые отряды
  сохраняются. Другие моды направляют мир. AlifeGuard держит его производительным.

MCM:
  Общие: лимиты популяции, буфер гистерезиса, интервал проверки, правила защиты
  (квестовые NPC, дальние первыми, поотрядная очистка, round-robin), уведомления КПК.
  Разработка: уровень логов (ERROR/WARN/INFO/DEBUG), Санитайзер смарт-террейнов
  (переключатель, интервал, кнопка немедленной очистки), диагностика, сброс популяции.

Установка (MO2):
1. Установите xlibs
2. Установите AlifeGuard
3. Порядок загрузки не имеет значения
4. Настройте через MCM

Удаление (MO2):
Отключите или удалите в MO2.

Совместимость:
Протестирован на ванильном Anomaly 1.5.3, Demonized main, Demonized MT и AOEngine (актуальные версии).
Не модифицирует базовые скрипты. Использует стандартный API движка (alife_release_id).
Совместим с конфигурационными твиками A-Life (настройки alife.ltx, max_population умных
территорий, Redone Alife Performance, x3 performance tweaks). Эти моды меняют параметры
движка, а AlifeGuard работает на уровне скриптов, поэтому они не пересекаются.
Отключите другие моды деспавна (например, Grok's Despawner) при использовании AlifeGuard.
Два губернатора популяции вызывают конфликтующие решения об удалении, циклы перерождения
и утечки сущностей.

Известная проблема:
Редкий краш при освобождении сущности (ассерт Perform_reject). Проблема на уровне
движка в отслеживании родителя инвентаря X-Ray, присутствует во всех модах популяции.
Скриптового исправления не существует.

Архитектура:
- AlifeGuard работает на xlibs, реверс-инженерном API, обёртывающем исходный код X-Ray.
  Жизненный цикл отрядов, проверки защиты и бухгалтерия спавна прослежены по исходному
  коду C++. Основные паттерны изучены по лучшим модам экосистемы Anomaly.
- Без правок базовых скриптов, без патчей движка. Только рантайм-коллбэки.
- Двухфазный конвейер: синхронный сбор (кадр 0), покадровое освобождение (кадры 1-N).
  См. doc/architecture.md для полной документации.

Производительность:
- Производительность была проектным ограничением с самого начала.
  Сбор: субмиллисекундный для 200 сущностей. Освобождение: 0.05мс за кадр.
- Кооперативное разделение времени (xslice) предотвращает каскадные краши net_Relcase,
  вызываемые пакетным освобождением в X-Ray.
- Структурированная трассировка с ID трасс, таймингами по фазам, синглтоны нулевых
  объектов для нулевой нагрузки отладки при уровне логов < DEBUG.

Многоступенчатый конвейер валидации:
- luacheck, selene (статический анализ)
- tree-sitter AST анализ, структурные паттерны ast-grep
- Контрактные правила (безопасность API, кросс-файловые зависимости, цикломатическая сложность, стандарты кода)
- Интеграционное тестирование lua54 со стабами движка X-Ray
- gitleaks (сканирование секретов)
Полный отчёт в doc/test-report.log.
